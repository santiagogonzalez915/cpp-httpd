Static file demo

This page and its assets (/style.css, /static/app.js) are served by StaticFileHandler with ETag validation and sendfile() zero-copy transfer.

Caching headers

Open DevTools → Network → reload this page. Select any response and look at its headers. You should see: 

ETag: "3a2f1c-2d4e-6b8f9a"   ← inode-size-mtime
Cache-Control: max-age=3600
Last-Modified: Mon, 16 Jun 2026 00:00:00 GMT

Reload again. The browser sends If-None-Match with the ETag. The server checks it, and if the file is unchanged, returns 304 Not Modified — no body transferred.

Range requests

The server supports Range: bytes=N-M (RFC 7233). Try fetching the first 64 bytes of this stylesheet: 

curl -H "Range: bytes=0-63" http://localhost:6789/style.css -i

You'll get a 206 Partial Content response with the exact byte slice.

sendfile()

Static file bodies are transferred with sendfile() — the kernel moves bytes from the page cache directly to the socket buffer without copying through user space. The server never reads the file into a heap buffer.
