How it works

AuthHandler looks for .htaccess in the directory of the requested resource. If found, it decodes the Authorization: Basic <base64> header, compares the credentials against the stored base64-encoded values, and returns 401 Unauthorized with a WWW-Authenticate header if they don't match — which is what triggered your browser's login prompt.

The .htaccess file

AuthType Basic
User YWRtaW4=      ← base64("admin")
Password ZGVtbw==  ← base64("demo")
The credentials for this demo are admin / demo. In a real deployment you'd use a stronger password and store a hash rather than base64 — but the auth flow is identical.
